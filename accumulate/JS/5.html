    # this

    首先this提供了一种更加优雅的方式来隐式“传递”一个对象的引用。

    • this并不指向函数本身
    • this指向函数的作用域，在某种情况下正确

    this在任何情况下都不指向函数的词法作用域，作用域“对象”无法通过JavaScript代码访问，它存在于JavaScript引擎内部。

    # this到底是什么？

    this是在运行时进行绑定的，它的上下文取决于函数调用时的各种条件。this的绑定和函数声明的位置没有任何关系。

    当一个函数被调用时，会创建一个活动记录（有时也叫执行上下文），这个记录会包含函数在哪儿被调用的（调用栈）、函数的调用方法、传入的参数等信息，而this就是记录中的一个属性，会在函数执行的过程中用到。

    # 绑定规则

    # 默认绑定

    即独立函数调用的情况，此时this指向全局对象。这里要区分一下非严格模式和严格模式strict mode：

    function foo(){
    "use strict"
    console.log(this.a)
}
var a = 2;
foo() // TypeError:this is undefined

    # 隐式绑定

    即调用位置是否有上下文对象，或者说被某个对象拥有或包含。

    function foo(){
    console.log(this.a)
}
var obj = {
    a:2,
    foo
}
obj.foo() // 2

    # 显式绑定

    我们可以用原型上提供的方法来进行显式绑定，即call apply,

    function foo(){console.log(this.a)}
var obj = {obj:a}
foo.call(obj) // 2
foo() // undefined

    # 硬绑定

    所谓硬绑定是显式绑定的一个变种，

    function foo(){
    console.log(this.a);
}
var obj = {
    a:2;
};
var bar = function(){
    foo.call(obj); // 只要调用bar就会强制绑定obj上
}
bar() // 2
setTimeout(bar,100) // 2
// 硬绑定的bar不可能再修改它的this
bar.call(window);

    # new 绑定

    在JavaScript中，new操作符的意义和其他面向对象的语言完全不同。而所谓的构造函数，本质上是普通的函数而已，只不过它们是被new操作符调用的。

    当我们使用new操作符时，它其实做了四件事：

    1. 创建（或者说构造）一个全新的对象。
    2. 这个新对象会被执行 [[ 原型 ]] 连接。
    3. 这个新对象会绑定到函数调用的 this。
    4. 如果函数没有返回其他对象，那么 new 表达式中的函数调用会自动返回这个新对象。

    new会构造一个新对象并将它绑定到函数调用中的this上。

    function foo(a){
    this.a = a;
};
var bar = new foo(2);
console.log(bar.a) //2

    # 判断this

    1. 函数被new调用：this绑定的是新创建的对象
    2. 函数通过apply call显式绑定：this绑定的是指定对象
    3. 函数在某个上下文对象中调用：this绑定这个上下文对象
    4. 都不是则默认绑定。
