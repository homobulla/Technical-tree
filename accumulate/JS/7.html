防抖和节流的概念很接近，二者都是将频繁的操作进行一定程度的缩减控制。

# debounce

函数防抖的基本思路是把高频信号化为一个信号。即会忽略多次触发，最终只会触发一次。

基本思路是通过定时器将回调函数进行延时，如果在规定时间内又触发回调，发现之前的定时器，则将该定时器清除，并重新设置定时器。

防抖的处理有两种方式，本质上来讲就是你决定将触发开关放在哪儿。即立即执行和最后触发。

  • 非立即执行：事件触发——> 延时——> 执行回调函数
  • 立即执行：事件触发——> 执行回调函数——> 延时
function debounce(func, delay=500) {
    var timeout;
    return function() {
        clearTimeout(timeout); 
        timeout = setTimeout(_=>{
          func.apply(this, arguments);
        },delay)
    };
};
// 处理函数
function handle() {
    console.log(Math.random())
}
// 滚动事件
window.addEventListener('scroll', debounce(handle, 1000))

# throttle

节流是在规定时间内执行一次。我们不会阻止它的多次触发，只是将触发它的频率降到我们自己定义的时间段内而已。所以我们需要记录上次执行和时间点和定时器。

//时间戳+定时器版: 实现第一次触发可以立即响应,结束触发后也能有响应 (该版才是最符合实际工作需求)
//该版主体思路还是时间戳版,定时器的作用仅仅是执行最后一次回调
function throttle(fun, delay = 500) {
     let timer = null;
     let previous = 0;
     return function(args) {
             let now = Date.now();
             let remaining = delay - (now - previous); //距离规定时间,还剩多少时间
             let that = this;
             let _args = args;
             clearTimeout(timer);  //清除之前设置的定时器
              if (remaining <= 0) {
                    fun.apply(that, _args);
                    previous = Date.now();
              } else {
                    timer = setTimeout(fun, remaining); //因为上面添加的clearTimeout.实际这个定时器只有最后一次才会执行
              }
      }
}
