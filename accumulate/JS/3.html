# 原型

# 原型和原型链

  • 所有的引用类型（数组、对象、函数），都具有对象特性，即可自由扩展属性（null 除外）
  • 所有的引用类型（数组、对象、函数），都有一个proto属性，属性值是一个普通的对象
  • 所有的函数，都有一个 prototype 属性，属性值也是一个普通的对象
  • 所有的引用类型（数组、对象、函数），proto属性值指向它的构造函数的 prototype 属性值

从上可知，Funtion是对象，Function.prototype也是一个对象，他们都具有对象的特点。即对象具有属性__proto__，可称为隐式原型。一个对象的隐式原型指向构造该对象的构造函数原型。

Function是一个特殊的对象，其具有一个独有的原型属性prototype，这个属性为一个指针，指向一个对象，这个对象的用途就是包含所有实例共享的属性和方法（此对象被称为原型对象）。这个原型对象同时也有一个属性叫constructor,这个constructor包含了一个指向原构造函数的指针。 而所谓的constructor并不表示被构造，且本身很容易被修改。构造函数也只是被new调用的函数而已。

显式原型是用来实现基于原型的继承与属性的共享，隐式原型则是用来构成原型链，同样用于实现基于原型的继承。

JS 正是通过__proto__和prototype合作实现了原型链以及对象的继承。

    function Foo(name){
        this.name = name;
    }
    Foo.prototype.sayName = function(){
        return this.name
    }
    function Bar(name,label){
        Foo.call(this,name);
        this.name = name;
    }
    //我们创建了一个新的 Bar.prototype 对象并关联到 Foo.prototype
    Bar.prototype = Object.create(Foo.prototype);
    // 这里的Object.create 会创建一个新的对象并将新对象内部的[prototype]指向关联的对象。
    // 注意！现在没有 Bar.prototype.constructor 了
    // 如果你需要这个属性的话可能需要手动修复一下它
    Bar.prototype.myLabel = function() {
    return this.label;
    };
    var a = new Bar( "a", "obj a" );
    a.myName(); // "a"
    a.myLabel(); // "obj a"
